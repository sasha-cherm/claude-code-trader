## Key Discovery: get_order_book() vs get_price()
- `get_order_book()` is MISLEADING for checking MM activity — it only shows resting CLOB orders
- Dynamic MMs don't rest orders on the book — they fill incoming orders on-the-fly
- **ALWAYS use `get_price()` to check if dynamic MMs are active** — if it returns a fair value (not 0.01/0.99), MMs are present
- This explains why previous sessions incorrectly flagged games as "404 orderbook" — the MMs were there all along
